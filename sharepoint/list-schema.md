# SharePoint List 欄位設計

清單名稱：`公務車借用管理`

用途：作為公務車借用後台管理資料來源，供承辦人確認借用人是否完成 Teams Adaptive Card 回覆，以及是否可發放鑰匙。

## 主要欄位

| 欄位名稱 | 用途 |
|---|---|
| 車輛名稱 | 顯示借用車輛 |
| 資源信箱 | 對應 Outlook Resource Mailbox |
| 行事曆事件 ID | 對應 Outlook 行事曆事件 |
| 預約唯一鍵 | 避免重複建立資料 |
| 借用日期 | 借用日期 |
| 借用起始時間 | 借用開始時間 |
| 借用結束時間 | 借用結束時間 |
| 借用人姓名 | 預約者姓名 |
| 借用人 Email | 預約者 Email |
| 使用事由 | Outlook 行事曆主旨 |
| 共乘人數 | 借用人回覆的共乘人數 |
| 規範確認狀態 | 是否同意公務車使用規範 |
| Teams 回覆時間 | 借用人送出卡片時間 |
| 通知發送時間 | Teams 通知送出時間 |
| 領鑰狀態 | 待通知／未完成填寫／已完成確認／已領鑰／取消 |
| 承辦人備註 | 行政／總務補充說明 |
| 最後同步時間 | Power Automate 最後更新時間 |

## 功能測試欄位

| 欄位名稱 | 用途 |
|---|---|
| 是否測試資料 | 是 = 功能測試資料；否 = 正式借用資料 |
| 功能測試結果 | 未測試／通過／未通過／待修正 |

## 上線原則

- 測試資料必須標記 `是否測試資料 = 是`。
- 正式啟用前，核心測試案例需為 `功能測試結果 = 通過`。
- 未完成填寫或未勾選規範確認時，`領鑰狀態` 不可更新為 `已完成確認` 或 `已領鑰`。
