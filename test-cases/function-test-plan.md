# 功能測試計畫

## 測試目標

確認「M365 公務車借用自動通知與後台管理流程」符合以下條件：

- Outlook 公務車資源行事曆資料可同步至 SharePoint。
- SharePoint 後台可即時查看借用狀態。
- Teams Adaptive Card 可於正確時間發送給借用人。
- 借用人回覆後可更新 SharePoint。
- 未完成填寫時，承辦人不得發放鑰匙。
- 全流程使用 O365 E1 可用標準功能，不使用 Premium 連接器。

## 測試資料原則

- 測試紀錄需勾選 `是否測試資料`。
- 測試完成後，`功能測試結果` 填入 `通過`、`未通過` 或 `待修正`。
- 測試資料不得與正式借用紀錄混淆。

## 測試通過標準

正式上線前，以下項目必須通過：

- 三台資源信箱皆可成功同步預約資料。
- 通知時間符合「借用前 1 小時，但最早 08:00」規則。
- 整天借用可於當天 08:00 通知。
- Teams Adaptive Card 可正確送達借用人。
- 借用人回覆可正確寫回 SharePoint。
- 未完整填寫時，`領鑰狀態` 不得變成 `已完成確認`。
- 承辦人可在 SharePoint 後台即時判斷是否發放鑰匙。
- 系統防呆測試 SG-T01 至 SG-T10 必須通過，才可正式上線。
