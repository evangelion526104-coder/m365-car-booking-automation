# 功能測試計畫

正式串接自動化前，需先完成此功能測試。

## 測試資料規則

- 測試資料在 SharePoint List 中需標記 `是否測試資料 = 是`。
- 測試尚未完成時，`功能測試結果 = 未測試`。
- 測試失敗時，`功能測試結果 = 未通過` 或 `待修正`。
- 測試成功後，`功能測試結果 = 通過`。

## 核心測試案例

| 編號 | 測試情境 | 預期結果 | 狀態 |
|---|---|---|---|
| FT-001 | Outlook 預約公務車 | SharePoint 建立一筆借用資料 | 未測試 |
| FT-002 | 借用前 1 小時通知 | Teams 收到 Adaptive Card | 未測試 |
| FT-003 | 填寫共乘人數並勾選規範 | SharePoint 狀態更新為已完成確認 | 未測試 |
| FT-004 | 未填共乘人數 | 狀態維持未完成填寫 | 未測試 |
| FT-005 | 未勾選規範確認 | 狀態維持未完成填寫 | 未測試 |
| FT-006 | Outlook 取消預約 | SharePoint 領鑰狀態更新為取消 | 未測試 |
| FT-007 | Outlook 修改時間 | SharePoint 起訖時間同步更新 | 未測試 |
| FT-008 | 同一台車重複預約 | Outlook 資源信箱阻擋或承辦人可辨識衝突 | 未測試 |
| FT-009 | 借用人未回覆 | 狀態維持未完成填寫，不可發放鑰匙 | 未測試 |

## 上線門檻

全部核心測試案例通過後，才能進入正式串接與正式通知。
