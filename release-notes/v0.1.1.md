# Release Note v0.1.1

日期：2026-06-25

## 版本重點

補上「依據專案不同流程階段完成狀態建立版本」的規則，確保每個階段成果都有獨立版本，且舊版本可以保留與追蹤。

## 本版新增

- 每個流程階段完成後都建立新版本。
- 舊版本使用 Git tag 保留，不覆蓋、不刪除。
- 每個版本需建立對應 release note。
- `CHANGELOG.md` 需集中記錄所有版本異動。
- 新增 `docs/stage-versioning-policy.md`。
- 更新 `README.md` 與 `docs/version-control-workflow.md`。

## 版本保留說明

- `v0.1`：保留為專案初始化版本。
- `v0.1.1`：新增階段版本與舊版本保留規則。

## 下一版預計

`v0.2`：完成 SharePoint List 階段版本，包括欄位設計、測試欄位與後台檢視。
