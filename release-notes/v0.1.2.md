# Release Note v0.1.2

日期：2026-06-25

## 版本重點

新增「不同流程階段完成時，Codex 自動進入 GitHub Releases 網頁建立新版本」的規則。

## 本版新增

- 新增 `docs/github-release-web-workflow.md`。
- 規定階段完成後不只建立 Git tag，也要建立 GitHub Release。
- 規定 GitHub Release description 需使用對應的 `release-notes/vX.X.md` 內容。
- 更新 README、CHANGELOG、階段版本政策與 Codex 檢查清單。

## 後續執行方式

未來只要某個階段完成，例如 SharePoint List 或 Adaptive Card：

1. Codex 更新文件與 CHANGELOG。
2. Codex commit 並 push。
3. Codex 建立並推送 tag。
4. Codex 自動開啟 GitHub Releases 網頁。
5. Codex 建立該版本 Release。
6. Codex 回報 Release 網址。

## 下一版預計

`v0.2`：完成 SharePoint List 階段版本，包括欄位設計、測試欄位與後台檢視。
